# Contributing

## Getting Started
- Fork/clone the repository.
- Install dependencies.
- Create a new branch from `main`.

## Branch Naming
Use the team branch naming convention:

team-alpha/<type>/<issue-short-description>

Example:

team-alpha/docs/53-contributing-guidelines

## Commit Messages

Follow conventional commits.

Examples:

docs: add contribution guidelines
fix: correct import path
chore: stop tracking generated output folders

## Pull Requests

- Open one PR per issue.
- Link the related issue.
- Keep PRs focused on a single change.
- Request at least one peer review.
- Request review from the required maintainer.
- Address review feedback before merging.

## Code Reviews

- Ensure acceptance criteria are met.
- Verify the change before requesting review.
- Keep discussions respectful and constructive.
